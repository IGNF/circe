function arguments => [parser] => circe options.

The options available to the users follow some of the POSIX recommendations. An option may be input by:

  • a long option name;
  • a long option name with an argument separated by a '=' without spaces;
  • a short option name;
  • a short option name with an argument with or without a separating space.

But as for multiple short option names without argument, unlike POSIX recommendations, ‘-abc’ is NOT equivalent to ‘-a -b -c’.

Hopefully the uppercase version of a lowercase key has a logical relation with it (it is often a source/target relation).
